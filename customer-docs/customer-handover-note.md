# Customer Handover Note

## Summary
Basic connectivity checks were completed on the test network. The device was checked for local IP settings, default route, internet reachability, DNS resolution and traceroute behaviour.

## Outcome
The main network connection was confirmed as working. Internet connectivity and DNS resolution were successful. Some diagnostic traffic, including gateway ping and later traceroute hops, appeared to be filtered by the guest Wi-Fi network.

## Basic Guidance for User
- Restart the router only after checking whether other devices are affected.
- Confirm the device is connected to the correct Wi-Fi network.
- If websites do not load, check whether the issue affects all websites or only one.
- Record any error messages before escalating the issue.
- Do not assume the network is down only because ping or traceroute fails.

## Escalation
If multiple devices lose connectivity, websites fail across all devices, or the router shows service errors, the issue should be escalated to the internet service provider or technical support team.
